# Day 2 - Data Structures: Lists and Dicts

A **list** is an ordered collection of items where position matters, you use it when you have a sequence of things of the same kind, like a list of scores or names, and you need to iterate over them or access them by index.

A **dict** is a collection of key-value pairs where each item has a meaningful label, you use it when every piece of data has a name or identity, like a student record where `name`, `scores`, and `subject` each mean something specific.

The rule of thumb: if you're thinking *"a bunch of X"*, reach for a list; if you're thinking *"this thing has properties"*, reach for a dict.
